# Документация по проекту: Интернет-магазин на FastAPI + Vue

Добро пожаловать в проект современного интернет-магазина (E-Commerce + Blog + IoT), построенного на базе модульной Clean Architecture.
Проект разрабатывается с помощью системы специализированных ИИ-агентов (на базе Gemini CLI), которые автоматизируют написание кода, тестирование, DevOps и аудит безопасности.

## Архитектура и стек технологий

Проект состоит из независимых модулей (товары, заказы, блог, IoT) и опирается на следующие технологии:
- **Backend**: Python 3.12+, FastAPI, SQLAlchemy 2.x (async), Alembic, PostgreSQL 16
- **Кэш и очереди**: Redis 7, Celery + Redis broker
- **Frontend**: Vue 3 + Pinia + Nuxt 3 (SSR для SEO блога)
- **Интеграции**: СДЭК v2 API, YooMoney (aiomoney), курсы валют ЦБ РФ
- **Инфраструктура**: Docker Compose, Nginx, Prometheus, Grafana, GitHub Actions

Более подробно архитектура и этапы описаны в файле [plan.md](plan.md).

## Разработка с помощью ИИ-агентов (Gemini CLI)

Проект использует многоагентную систему. Вместо ручного написания кода вы ставите задачи оркестратору или конкретным агентам.

### Доступные агенты
- `orchestrator` — управляет задачами, делегирует их другим агентам и проверяет отчеты.
- `backend-agent` — пишет API (FastAPI, SQLAlchemy, Pydantic).
- `frontend-agent` — разрабатывает UI (Vue 3, Nuxt 3, Pinia).
- `testing-agent` — покрывает код тестами (pytest, respx, Locust).
- `devops-agent` — настраивает инфраструктуру (Docker, Nginx, CI/CD).
- `security-agent` — проводит аудит безопасности (Read-Only).
- `cdek-agent` — реализует интеграции СДЭК и YooMoney.

### Как ставить и запускать задачи

Вы можете управлять агентами через специальные пользовательские команды Gemini CLI:

1. **Создание (постановка) задачи:**
   Используйте команду `/agents:start`, указав имя агента и описание задачи.
   ```bash
   /agents:start <agent-name> <описание задачи>
   ```
   *Пример:* `/agents:start backend-agent Создать эндпоинты для каталога товаров согласно plan.md`
   Это создаст JSON-файл задачи в директории `.gemini/agents/tasks/`.

2. **Запуск задачи из очереди:**
   Используйте команду `/agents:run` для выполнения задач.
   ```bash
   /agents:run [task_id]
   ```
   Если `task_id` не указан, будет запущена самая старая задача со статусом "pending". Агент автоматически загрузит свои инструкции, выполнит код, проверит контракты (линтеры/тесты) и сгенерирует отчет.

3. **Проверка статуса:**
   Используйте команды статуса (например, `/agents:status`), чтобы проверить состояние задач и агентов.

### Контракты и отчеты

Разработка ведется по строгим правилам (Design-by-Contract):
- Все API эндпоинты должны иметь Pydantic-схемы.
- Сервисы используют Dependency Injection.
- Внешние вызовы защищены паттерном retry (tenacity).
- По завершении работы каждый агент **обязан** сформировать markdown-отчет в директории `.gemini/agents/reports/<domain>/<task_id>.md`, содержащий статус, список артефактов и проверенные контракты.

## Запуск проекта (Сборка)

Инфраструктура проекта поднимается через Docker Compose. За это отвечает `devops-agent`.
Для локального запуска (после того как `devops-agent` создаст конфигурацию):

```bash
docker-compose up --build
```
Это поднимет:
- `api` (FastAPI на 8000 порту)
- `postgres` (база данных)
- `redis` (кэш и брокер)
- `celery_worker` и `celery_beat` (фоновые задачи)
- `frontend` (Vue 3 / Nuxt 3)
- `nginx` (Reverse Proxy)

Все секреты (пароли, API ключи) должны быть определены в файле `.env` (без хардкода!).
