# Документация по проекту: Интернет-магазин на FastAPI + Vue

Добро пожаловать в проект современного интернет-магазина (E-Commerce + Blog + IoT), построенного на базе модульной Clean Architecture.
Проект разрабатывается с помощью системы специализированных ИИ-агентов (на базе Gemini CLI).

## Архитектура и стек технологий

Проект состоит из независимых модулей (товары, заказы, блог, IoT) и опирается на следующие технологии:
- **Backend**: Python 3.12+, FastAPI, SQLAlchemy 2.x (async), Alembic, PostgreSQL 16
- **Кэш и очереди**: Redis 7, Celery + Redis broker
- **Frontend**: Vue 3 + Pinia + Nuxt 3
- **Инфраструктура**: Docker Compose, Nginx, Prometheus, Grafana

## Работа с ИИ-агентами (Gemini CLI)

Агенты теперь интегрированы непосредственно в Gemini CLI как под-агенты (sub-agents). Вы можете обращаться к ним напрямую в чате, используя символ `@`.

### Доступные агенты
- `@orchestrator` — Управляющий агент.
- `@backend-agent` — Разработка серверной части.
- `@frontend-agent` — Разработка клиентской части.
- `@testing-agent` — Написание тестов.
- `@devops-agent` — Настройка инфраструктуры.
- `@security-agent` — Аудит безопасности.
- `@cdek-agent` — Интеграция СДЭК и YooMoney.

### Способы вызова

1. **Прямой вызов в чате:**
   Вы можете просто написать: `@orchestrator создай план реализации корзины`.

2. **Через систему задач (автоматизация):**
   - `/agents:start <agent-name> <задача>` — поставить задачу в очередь.
   - `/agents:run [task_id]` — запустить выполнение задачи из очереди.

### Конфигурация

Определения агентов находятся в `.gemini/agents/*.md`. Для их работы в `.gemini/settings.json` включена опция `enableAgents: true`.
