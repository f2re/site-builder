# Tasks — инструкции для агентов

Каждый файл — задача для конкретного агента. Имя файла = ID задачи.

## Конвенция именования

```
<YYYYMMDD>_<этап>_<агент>_<домен>.md
```

## Статусы

| Статус | Значение |
|--------|----------|
| `TODO` | Не начата |
| `IN_PROGRESS` | Агент работает |
| `DONE` | Завершена, отчёт в `reports/` |
| `BLOCKED` | Заблокирована зависимостью |

## Правило удаления

**Задачи со статусом `DONE` НЕ удаляются** — они архив принятых решений.
Агент-оркестратор читает их чтобы не дублировать работу.
Если задача устарела и не нужна — переводи в `CANCELLED` с причиной.

## Очерёдность (приоритет)

1. `BE-01` → `BE-02` (модели нужны раньше API)
2. `BE-03` зависит от `BE-01` (корзина → продукты)
3. `BE-04` и `FE-01..03` можно запускать параллельно после `BE-01`
4. `BE-05` (IoT) независим — можно параллельно
5. `FE-04` (SEO) — только после `FE-01..03`
6. `QA-01` — в конце всего
