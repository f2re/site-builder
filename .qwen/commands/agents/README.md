# Slash-команды для QWEN агентов

## /agents:plan <описание задачи>

**Описание:** Декомпозиция задачи на подзадачи для агентов.

**Использование:**
```
/agents:plan Добавить функционал корзины с Redis
/agents:plan Реализовать WebSocket для IoT телеметрии
/agents:plan Настроить CI/CD пайплайн для production
```

**Что делает оркестратор:**
1. Анализирует задачу
2. Определяет необходимых агентов
3. Создаёт `.json`-файлы задач в `.qwen/agents/tasks/`
4. Выводит план выполнения с зависимостями

---

## /agents:status

**Описание:** Показать статус текущих задач и агентов.

**Вывод:**
- Активные задачи по агентам
- Прогресс выполнения фаз
- Блокеры (если есть)

---

## /agents:report <task_id>

**Описание:** Получить отчёт по выполненной задаче.

**Использование:**
```
/agents:report p1_backend_001
```

---

## /agents:validate

**Описание:** Запустить финальную валидацию проекта (Gatekeeper).

**Что проверяется:**
- `alembic heads` — один head
- `alembic check` — синхронизация моделей
- `ruff check` / `mypy` — линтинг backend
- `npm run lint` — линтинг frontend
- `pip install -r requirements.txt` — зависимости
- `pytest` — тесты

---

## /agents:contract <domain>

**Описание:** Показать контракты для домена.

**Использование:**
```
/agents:contract api
/agents:contract auth
/agents:contract iot
/agents:contract theme
```

---

## /shop:init

**Описание:** Инициализировать новую фазу разработки магазина.

**Что делает:**
1. Создаёт структуру задач для фазы
2. Проверяет зависимости от предыдущих фаз
3. Запускает devops-agent для подготовки инфраструктуры

---

## /blog:init

**Описание:** Инициализировать новую фазу разработки блога.

---

## /iot:init

**Описание:** Инициализировать новую фазу разработки IoT-дашборда.

**Особенности:**
- Требует готовой инфраструктуры (TimescaleDB, Redis Streams)
- Запускает backend-agent для WebSocket + телеметрии
- Запускает frontend-agent для дашборда

---

## /admin:init

**Описание:** Инициализировать новую фазу разработки админ-панели.

**Требования:**
- Все CRUD эндпоинты должны быть готовы
- Role-based access control реализован
