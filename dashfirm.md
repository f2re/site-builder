# Техническое задание на разработку модуля «Dashfirm» (управление прошивками)

## 1. Общее описание
Модуль предназначен для управления базой данных устройств (по серийным номерам) и привязки их к пользователям через уникальные токены. Основная функция — генерация и выдача файлов прошивок (.bin) на основе выбранной версии и конфигурации (комплектации) для конкретного серийного номера через внешний компилятор.

---

## 2. Схема базы данных (Сущности и связи)

### 2.1. Таблица токенов (`module_tokens`)
Служит «ключом доступа» пользователя к списку его устройств.
- `id`: Первичный ключ.
- `token`: Уникальная строка (рекомендуется hex, 40-64 символа). Генерируется криптографически: `bin2hex(random_bytes(20))`.
- `customer_id`: Связь с таблицей пользователей системы. У одного пользователя может быть только один активный токен.

### 2.2. Таблица оборудования (`module_token_to_device`)
Связующая таблица между токеном и физическим оборудованием.
- `id`: Первичный ключ.
- `token_id`: Внешний ключ к таблице токенов.
- `serial`: Серийный номер устройства (уникальный индекс).
- `device_type`: Тип устройства (ENUM: `OBD`, `AFR`).
- `comment`: Текстовое примечание администратора.
- `date_added`: Дата добавления записи.

### 2.3. Справочник комплектаций (`module_complectations`)
- `id`: Первичный ключ.
- `caption`: Понятное название (напр., «Базовая + ISO»).
- `label`: Системный идентификатор (напр., `base_iso`). Используется при вызове компилятора.
- `code`: Числовой код (битовая маска или уникальный ID для сумматора: 1, 2, 4, 8...).
- `simple`: Флаг (0/1).
    - `1`: Простая опция (чекбокс в админке).
    - `0`: Итоговая сборная конфигурация (результат сложения кодов).

### 2.4. Комплектации устройств (`module_complectation_to_device`)
Какие опции разрешены администратором для конкретного серийного номера.
- `serial`: Серийный номер устройства.
- `complectation_id`: ID из справочника комплектаций.

---

## 3. Логика административной панели

### 3.1. Управление устройствами
- **Добавление**: При вводе серийного номера система проверяет наличие токена у пользователя. Если его нет — генерирует автоматически.
- **Принудительная база**: Каждому новому устройству автоматически назначается базовая комплектация (`label = 'base'`).
- **Поиск**: По серийному номеру, токену или ФИО/Email пользователя.

### 3.2. Управление дубликатами (Merge)
При слиянии пользователя А (старый) в пользователя Б (новый):
1. Все устройства из `module_token_to_device` переносятся на `token_id` пользователя Б.
2. Если серийник уже есть у пользователя Б, дубликат от А удаляется.
3. Токен пользователя А аннулируется.

### 3.3. Импорт данных (Excel/CSV)
Формат файла предполагает два листа:
- **Лист 1 (Клиенты)**: `temp_id`, ФИО, контакты.
- **Лист 2 (Устройства)**: `temp_id`, `serial`, `complectation_code`, `type`.
Парсер сначала создает пользователей, запоминает соответствие `temp_id -> real_id`, затем привязывает устройства.

## 4. Механизм «Сумматора» комплектаций

Итоговая прошивка определяется суммой кодов выбранных пользователем опций.

**Алгоритм:**
1. Пользователь на фронтенде выбирает чекбоксы разрешенных ему опций (`simple = 1`).
2. Система суммирует значения их поля `code`.
3. Система ищет в таблице `module_complectations` запись, где `code` равен этой сумме И `simple = 0`.
4. Полученный `label` этой записи передается в компилятор.

---

## 5. Взаимодействие с компилятором (CLI)

Все вызовы должны использовать экранирование аргументов (`escapeshellarg`).

### 5.1. Получение списка версий
`cd [path] && ./compiler [device_type] revisions`
Команда возвращает строку с доступными ревизиями, которую нужно распарсить в массив.

### 5.2. Сборка прошивки (Make)
`cd [path] && ./compiler [device_type] make [serial] [version] [final_label]`
- **Путь к файлу**: Компилятор сохраняет результат в подпапку `build/`.
- **Имя файла**: `Firmware_{TYPE}_{VERSION}_{LABEL}_{SERIAL}.bin`.

---

## 6. Безопасность и выдача файлов

1. **Доступ**: Файлы прошивок должны храниться вне публичной директории (`public_html`).
2. **Скачивание**: Реализуется через контроллер, который проверяет:
    - Авторизован ли пользователь.
    - Принадлежит ли запрашиваемый серийный номер его токену.
3. **Заголовки**: При отдаче использовать `Content-Type: application/octet-stream` и `Content-Disposition: attachment`.

## 7. Примеры кода (Псевдокод)

### Расчет итогового Label
```php
$total_code = $db->query("SELECT SUM(code) FROM module_complectations WHERE id IN ($selected_ids)");
$final_label = $db->query("SELECT label FROM module_complectations WHERE code = $total_code AND simple = 0");
```

### Вызов компилятора (PHP)
```php
$cmd = sprintf("cd %s && ./%s %s make %s %s %s",
    escapeshellarg($compiler_dir),
    escapeshellarg($binary),
    escapeshellarg($type),
    escapeshellarg($serial),
    escapeshellarg($version),
    escapeshellarg($final_label)
);
exec($cmd);
```

## 8. Пользовательское взаимодействие и интерфейс (Frontend)

Пользователь работает с системой через выделенную страницу «Программное обеспечение». Взаимодействие делится на три ключевых этапа: идентификация, настройка прошивки и управление устройствами.

### 8.1. Идентификация и доступ
Существует два способа получить доступ к списку своих устройств:
1.  **Поиск по токену**: Пользователь вводит свой уникальный 40-символьный ключ в поле поиска. Если токен найден, страница перезагружается и отображает список привязанных серийных номеров.
2.  **Личный кабинет**: Если пользователь авторизован, система автоматически подтягивает его токен и отображает список оборудования.

### 8.2. Выбор и настройка прошивки (Рабочая область)
Для каждого отображаемого устройства (серийного номера) доступна панель управления прошивкой:

1.  **Выбор версии (Dropdown)**:
    - При изменении версии в выпадающем списке срабатывает AJAX-запрос (`getVersionInfo`).
    - Система динамически подгружает и отображает:
        - **Changelog**: Список изменений в данной ревизии.
        - **Ссылки на документацию**: Ссылки на Web-интерфейс, WiFi-мост и другие компоненты, если они прописаны в `versionInfo.json`.
    - Во время загрузки данных отображается лоадер (spinner) поверх таблицы характеристик.

2.  **Выбор опций (Complectation Toggle)**:
    - Отображается набор кнопок-переключателей (например: "ISO", "CAN", "K-Line").
    - Кнопки активны только если администратор разрешил эти опции для данного серийного номера (таблица `module_complectation_to_device`).
    - Пользователь кликает по кнопкам, визуально выделяя нужные. При клике скрытое поле формы (`complect-SERIAL[]`) заполняется ID выбранной опции.

3.  **Запуск скачивания**:
    - При нажатии кнопки «Скачать» отправляется POST-форма, содержащая: `token`, `download_serial`, `version` и массив выбранных `complect[]`.
    - Сервер выполняет проверку прав, рассчитывает итоговый `label` (через сумматор), вызывает компилятор и отдает готовый `.bin` файл в браузер.

### 8.3. Управление устройствами
Пользователь может самостоятельно управлять своим списком оборудования:
- **Добавление нового устройства**: Кнопка «Добавить устройство» открывает форму ввода серийного номера. После сохранения устройство привязывается к текущему токену.
- **Редактирование**: Изменение типа устройства (OBD/AFR) или добавление личного комментария к серийному номеру.
- **Удаление**: Возможность отвязать серийный номер от своего токена.

### 8.4. Динамические эффекты (UX)
- **Pulse-эффект**: При AJAX-загрузке новых данных о версии таблица плавно подмигивает (loading-pulse).
- **Инвалидация кнопок**: Если опция недоступна для данного серийника, кнопка отображается в состоянии `disabled` и не реагирует на клики.
- **Информационные уведомления**: После добавления или изменения устройства пользователь получает Flash-уведомление о статусе операции.
