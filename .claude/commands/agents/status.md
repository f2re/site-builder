# /agents:status

> Показывает статус всех активных задач и отчётов агентов
> Использование: `/agents:status`
> Аргументы не требуются.

---

Выполни следующее:

## ШАГ 1 — Считай все задачи

Прочитай все файлы в `/.claude/agents/tasks/`.
Для каждого JSON запомни: `task_id`, `phase`, `agent`, `title`, `depends_on`, `priority`.

## ШАГ 2 — Считай все отчёты

Прочитай все `.md`-файлы в `/.claude/agents/reports/` (включая поддиректории `backend/`, `frontend/`, `testing/`, `devops/`).
Для каждого отчёта запомни статус из строки `## Status:`.

## ШАГ 3 — Сопоставь задачи с отчётами

Для каждой задачи определи статус:
- ✅ **DONE** — есть отчёт с `Status: DONE`
- ⚠️ **BLOCKED** — есть отчёт с `Status: BLOCKED`
- 🟡 **IN_PROGRESS** — есть отчёт с `Status: IN_PROGRESS`
- ⏳ **PENDING** — отчёт отсутствует, задача ещё не стартована
- 🔒 **BLOCKED (deps)** — зависимость не выполнена

## ШАГ 4 — Определи незаблокированные задачи

Задача разблокирована (готова к запуску), если:
- Статус: **PENDING** или **IN_PROGRESS**
- И все `depends_on` имеют `Status: DONE`

## ШАГ 5 — Выведи дашборд

Выведи сводную таблицу:

```
───────────────────────────────────────────────────────────
 📋 /agents:status — WifiOBD Site
───────────────────────────────────────────────────────────
Фаза  Агент              task_id              Статус     Приоритет
───────────────────────────────────────────────────────────
  1    devops-agent        p1_devops_001        ✅ DONE     high
  2    backend-agent       p2_backend_001       ⏳ PENDING  high
  ...
───────────────────────────────────────────────────────────
 🟢 Готовы к запуску (разблокированы):
  - p2_backend_001 [backend-agent] — <название>

 ⚠️ Блокеры:
  - p3_frontend_001: зависит от p2_backend_001 (не DONE)

 ⚠️ Требует вмешательства человека:
  - (none / описание блокера)
```

## ШАГ 6 — Обнови сводный отчёт

Запиши / обнови `/.claude/agents/reports/orchestrator_summary.md` со содержимым:

```markdown
# Orchestrator Summary
Обновлено: <дата время>

## Текущая фаза: <N>
## Выполнено задач: X / Y

| task_id | Агент | Титул | Статус |
|---|---|---|---|
| p1_devops_001 | devops-agent | ... | ✅ DONE |
| p2_backend_001 | backend-agent | ... | ⏳ PENDING |

## Блокеры:
- none

## Следующий шаг:
- /agents:run backend-agent p2_backend_001
```
